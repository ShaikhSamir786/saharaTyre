# Sahara Tyre – Tyre Shop in Dubai

A professional, mobile-first tyre shop website for Sahara Tyre in Dubai. This project features a modern single-page design, clear service presentation, and easy contact options.

## 🚀 Features

- **Modern Single-Page Design** – Clean, fast, and easy to navigate.
- **Mobile-First** – Looks perfect on all devices (phones, tablets, desktops).
- **Clear Service Presentation** – Highlights key services: New Tyres, Used Tyres, Wheel Alignment, Puncture Repair.
- **Easy Contact** – Click-to-call, WhatsApp button, and location map.
- **SEO Optimized** – Includes `sitemap.xml` and `robots.txt` for better search engine visibility.

## 📂 Project Structure

```
saharaTyre/
├── homepage.htm          # Main landing page
├── styles.css            # Styles and animations
├── script.js             # Interactive features
├── sitemap.xml           # XML sitemap for search engines
├── robots.txt            # Robots.txt for search engine crawlers
├── README.md             # Project documentation
└── vercel.json           # Vercel deployment configuration
```

## 🛠️ Getting Started

### Prerequisites

- A modern web browser.
- (Optional) Node.js and npm for local development.

### Local Development

1. Clone the repository:
   ```bash
   git clone <repository-url>
   cd saharaTyre
   ```

2. Open `homepage.htm` in your browser:
   ```bash
   open homepage.htm  # macOS
   start homepage.htm # Windows
   ```

## 🚀 Deployment

This project is ready for deployment to Vercel.

1. Create a Vercel account if you don't have one.
2. Import the project into Vercel:
   ```bash
   vercel --prod
   ```

Vercel will automatically detect the project and deploy it.

## 📝 SEO Configuration

- **Sitemap**: `https://sahara-tyre.vercel.app/sitemap.xml`
- **Robots.txt**: `https://sahara-tyre.vercel.app/robots.txt`

These files are automatically generated and updated.

## 📱 Testing

### Mobile Preview

Use your browser's developer tools to test the mobile experience:

1. Open `homepage.htm`.
2. Right-click and select "Inspect" or press `F12`.
3. Click the "Toggle device toolbar" icon (looks like a phone/tablet).
4. Select different devices (iPhone, Pixel, iPad) to test responsiveness.
